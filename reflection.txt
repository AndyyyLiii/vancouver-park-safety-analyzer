I built the Vancouver Park Safety Analyzer, a Python project that combines City of Vancouver park data with VPD crime data to study how parks and neighbourhood safety relate. The project loads and cleans both datasets, generates a printed text report with summary statistics and cross-dataset insights, produces three static PNG charts, and creates an interactive Folium map. Everything runs from one entry point: python main.py. The map is deployed via GitHub Pages so anyone can explore it without installing Python.

The hardest part was the cross-dataset chart. My first attempt was a scatter plot of washroom coverage versus crime count per neighbourhood, but the Central Business District was such an extreme outlier that it flattened every other point onto the bottom of the chart, and the labels overlapped badly. I then tried a grouped bar chart that averaged facility coverage across crime tiers, but it felt too aggregated and lost the story. Next I tried a dual-axis bar-and-line chart, which looked cleaner but still didn't reveal a clear pattern. Finally I switched to a hexbin geographic heatmap with park markers overlaid, which actually told a story I could explain in one sentence. To draw it I had to convert park lat/lon into the same UTM grid the VPD crime data uses, which I did with a flat-earth linear approximation to avoid adding pyproj as a dependency. I also discovered that around 9% of crime records have zero coordinates because VPD censors them for privacy, and I had to filter these out.

GitHub setup was harder than I expected. Git wasn't on my system PATH at first, I accidentally initialized the repo in the wrong parent folder and had to redo it, and at one point I deleted the wrong charts folder while cleaning up. I also hit a 403 error when previewing the map locally because OpenStreetMap tile servers now require a Referer header, so I switched to CartoDB Voyager tiles.

If I did this again, I would think about the cross-dataset story before writing chart code, instead of iterating through four versions. I would also set up a Git repository and a .gitignore from day one of Milestone 1, not at the very end. Finally, I would explore the raw CSVs more carefully up front to catch issues like the zero-coordinate records before they affected my charts.
